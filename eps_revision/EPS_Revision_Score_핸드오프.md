# EPS Revision 서브스코어 — 개발 핸드오프 문서

> **목적**: "롱숏 알파 스코어"의 EPS 항목(종합 = **EPS×35** + 상대강도×20 + 이벤트×12 + 퀄리티×10)을 계산하는 모듈을 만든다.
> **산출물**: 개별 종목에 대해 `-100 ~ +100` 섹터상대 점수 + 레이어별 분해 + 근거(evidence) + 한 줄 인사이트 + 정합성 플래그.
> **이 문서를 읽는 순서**: ①설계 철학 → ②3-레이어 구조 → ③집계·안전장치 → ④빌드 플랜(단계별 프롬프트) → ⑤주의사항.

---

## 1. 설계 철학 (왜 이렇게 만드는가)

**한 줄 요지: "재무 숫자의 수준"이 아니라 "컨센서스가 변하는 방향·속도·앞으로 더 변할 압력"을 산다.**

- 절대 실적값(매출 333조, OP 43조)은 **이미 주가에 반영**돼 있어 알파가 거의 없다.
- 알파는 **애널리스트 추정치가 바뀌는 과정**에 있다. 애널리스트는 새 정보에 한 번에 반응하지 않고 수개월에 걸쳐 추정치를 점진적으로 조정한다(**언더리액션**).
- 그래서 리비전은 **양(+)의 자기상관(관성)** 을 갖는다 → "최근 강하게 상향된 종목이 앞으로도 더 상향될 확률이 높다."
- **이 관성을 먹는 것이 EPS Revision 팩터의 본질이다.**

> ⚠️ 절대 규칙: 점수의 입력은 **컨센서스 추정치의 변화율**(`현재 / 3M전 - 1`)이다. **YoY 성장률(전년 실적 대비)은 절대 점수에 쓰지 않는다.** (사유는 5번 참고)

---

## 2. 3-레이어 구조

### Layer 1 — 실현 리비전 (이미 일어난 변화, backward)
변화가 실제로 일어났음을 확인한다.

| 컴포넌트 | 정의 | 의미 |
|---|---|---|
| `rev_op_3m`, `rev_eps_3m` | `추정치_now / 추정치_3M전 - 1` | 컨센 상향 크기 |
| `rev_op_1m`, `rev_eps_1m` | 1M 버전 | 최근 변화 |
| `diffusion_idx` | `(상향수 - 하향수) / 전체` | 변화의 **폭** (한두 명인지 컨센 전반인지) |
| `sue` | 최근 4Q 평균 `(actual - consensus)/|consensus|` | 어닝 서프라이즈 (추가 상향 선행) |

### Layer 2 — 리비전 모멘텀 (변화의 2차 미분: 가속/감속)
관성이 살아 있는지 본다.

| 컴포넌트 | 정의 | 의미 |
|---|---|---|
| `accel` | `(rev_1m × 12) − (rev_3m × 4)` | 상향 속도가 빨라지는 중인가 |
| `diffusion_trend` | 상향 비율의 개선 여부 | 폭이 넓어지는가 |
| `disp_cv` | `std / |mean|` | 추정치 분산. **축소 = 컨센 수렴 = 신뢰도↑** |

### Layer 3 — 포워드 압력 (forward, ★핵심)
미래를 **점치지 않는다.** 공식 컨센 변경을 **역사적으로 선행하는 관측 가능한 신호**를 모은다.

| 컴포넌트 | 정의 | 의미 |
|---|---|---|
| `runrate_gap` | `YTD누적 / (연간컨센 × 경과분기/4) − 1` | YTD가 런레이트 상회 → **컨센이 기계적으로 따라 올려야 함** (가장 깨끗한 forward 신호) |
| `tp_lead` | `(TP_now/TP_3M전 − 1) − rev_eps_3m` | 목표주가가 EPS보다 먼저 올랐나 → EPS 상향 선행 |
| `persistence` | `섹터_리비전_자기상관계수 × rev_op_3m` | 관성(언더리액션)의 직접 캐리. **자기상관계수는 외부 주입** |
| `news_lead` | KR-FinBERT 점수 | 보조 신호 (가중 낮게) |

## 3. 집계 · 안전장치

**집계 순서**
1. 각 컴포넌트를 **섹터 내 robust z-score**로 표준화 (winsorize ±3σ 선적용)
2. 레이어 가중합: **실현 0.40 · 모멘텀 0.25 · 포워드 0.35** (forward 강조)
3. 레이어 내부 컴포넌트는 균등가중에서 시작 (추후 튜닝)
4. **× 신뢰도 멀티플라이어**
5. 섹터 내 percentile → `-100 ~ +100` 스케일

**안전장치 1 — 신뢰도 게이트 (0.5 ~ 1.0, 점수에 곱함)**
- `analyst_n < 3` → 강한 캡 (점수 신뢰 불가)
- 추정 평균 경과일 길수록 디스카운트 (스테일 추정)
- `fy_roll_flag` (FY1→FY2 롤오버 구간) → 디스카운트 (가짜 리비전 발생)
- 분산 과대 → 디스카운트

**안전장치 2 — 베이스 효과 필터 (정합성 플래그)**
- 화면 사례: 예상 OP 85.5조 vs 2025년 43.6조 → 실제 YoY ≈ **+96%** 인데 화면엔 **+1728.7%** 로 표기됨.
- 원인: 2023년 트로프(OP 6.6조)를 베이스로 삼은 착시. → **점수에 반영 금지 + 플래그 + 인사이트에 명시.**

## 4. 빌드 플랜 — Claude Code에 줄 단계별 프롬프트

> **작업 방식**: 한 번에 전체를 시키지 말 것. 아래 STEP을 **하나씩** 복사해 붙이고, 단위테스트 통과를 확인한 뒤 다음 STEP으로. 스캐폴드 파일(`eps_revision_score.py`)을 먼저 열어 구조를 파악할 것.
> **팁**: STEP 0에 실제 FnGuide/Bloomberg 응답 샘플(한 종목 JSON)을 붙이면 스키마 정확도가 크게 오른다.

### STEP 0 — 컨텍스트 세팅 (한 번만)
```
개별 종목의 "EPS Revision 서브스코어" 계산 모듈을 만든다. 이 점수는 상위 종합스코어(종합 = EPS×35 + 상대강도×20 + 이벤트×12 + 퀄리티×10)의 EPS 항목으로 들어간다.

핵심 철학: 재무지표 절대값이 아니라 (1) 이미 일어난 컨센서스 리비전, (2) 그 변화의 가속/감속, (3) 미래 리비전 압력 — 이 3개 레이어로 분해해 점수화한다. 출력은 -100~+100 섹터상대 점수 + 레이어별 분해 + 근거 dict + 한 줄 인사이트.

전체 구조를 다음 순서로 함께 만들 거다: Layer1(실현리비전) → Layer2(모멘텀) → Layer3(포워드압력) → 신뢰도게이트 → 섹터 표준화·집계 → 인사이트 생성. 한 번에 다 만들지 말고 내가 단계별로 지시하면 그 단계만 구현해라.

기술 제약: Python/pandas, 순수 함수로 작성, 외부 I/O는 분리(데이터 주입식). 섹터 상대화는 별도 함수로 단일종목/배치 둘 다 지원. 첨부한 eps_revision_score.py 스캐폴드의 구조와 데이터 스키마를 그대로 따른다.

먼저 스캐폴드를 읽고, 데이터 스키마(dataclass)와 모듈 구조를 확인했다고 알려줘. 아직 계산 로직은 채우지 마.
```

### STEP 1 — Layer 1 (실현 리비전) + 테스트
```
이제 Layer1(실현 리비전) 함수만 구현해줘.
- rev_op_3m = op_fy1_now / op_fy1_3m_ago - 1  (반드시 추정치 변화율. YoY 성장률 절대 사용 금지)
- rev_op_1m, rev_eps_1m, rev_eps_3m 동일
- diffusion_idx = (up_count - down_count) / total
- sue = 최근 4Q 평균 (actual_op - consensus_op) / |consensus_op|
raw 값 반환 순수 함수. 결측치는 None 반환(0으로 채우지 말 것). 분모 0/음수 가드 포함.
단위테스트: (1)정상 상향 (2)분모 0 (3)결측. pytest로 작성하고 통과시켜줘.
```

### STEP 2 — Layer 2 (모멘텀)
```
Layer2(리비전 모멘텀=변화의 2차 미분) 함수를 구현해줘.
- accel = (rev_op_1m * 12) - (rev_op_3m * 4)   (양수=상향 가속)
- diffusion_trend = diffusion_idx 개선 여부(이전 분기 데이터 있으면)
- disp_cv = dispersion.std / |dispersion.mean| (축소=컨센 수렴=양호). 현재 CV 우선 반환.
raw 값 반환 순수 함수 + 결측 None + 가드. 단위테스트: 가속/감속/결측.
```

### STEP 3 — Layer 3 (포워드 압력) ★핵심
```
Layer3(미래 리비전 압력) 함수를 구현해줘. 미래를 점치는 게 아니라 공식 컨센 변경을 역사적으로 선행하는 신호를 모은다.
- runrate_gap = ytd_cumulative_op / (fy_consensus_op * quarters_elapsed / 4) - 1
- tp_lead = (tp_now / tp_3m_ago - 1) - rev_eps_3m
- persistence = sector_revision_autocorr(외부 주입, 0~1) * rev_op_3m   (모듈 내부에서 추정 금지)
- news_lead = news_sentiment (보조, 가중 낮게)
raw 값 반환 순수 함수 + 결측 None + 가드. 단위테스트: 런레이트 상회/하회, TP선행, 결측.
```

### STEP 4 — 신뢰도 게이트
```
신뢰도 멀티플라이어(0.5~1.0)를 산출하는 함수를 구현해줘. 최종 점수에 곱한다.
- analyst_n < 3 이면 강한 캡(0.5~0.6), 25개사 이상이면 1.0 수렴(구간별 선형 또는 sigmoid)
- avg_estimate_age_days 길수록 디스카운트(90일 초과 감점)
- fy_roll_flag == True 면 디스카운트
- dispersion CV 비정상 과대 시 디스카운트
각 요소를 0~1 페널티로 만들어 곱하고 0.5~1.0로 클립. 단위테스트: 풀신뢰(25개사 신선)/저커버리지(2개사)/롤오버.
```

### STEP 5 — 섹터 표준화 + 집계
```
aggregate.py를 구현해줘. Layer1/2/3 raw 컴포넌트를 섹터 내 표준화 후 합산한다.
- 각 컴포넌트를 섹터 내 robust z-score로 표준화(winsorize ±3σ 선적용)
- 레이어 가중: realized 0.40 · momentum 0.25 · forward 0.35
- 레이어 내부 컴포넌트는 균등가중 시작
- × 신뢰도 멀티플라이어(STEP4)
- 섹터 내 percentile → -100~+100 매핑
단일종목 모드 + 배치 모드 둘 다 지원. 결측 컴포넌트는 가중 0으로 빼고 나머지 재정규화(0으로 채우지 말 것).
단위테스트: 배치 5종목/단일종목/일부 결측.
```

### STEP 6 — 정합성 가드 + 인사이트 + 최종 출력
```
마지막으로 구현해줘.
(A) 정합성 가드: YoY 정합성 체크 — |fy_consensus_op / 직전연도_actual_op - 1| 계산해, 표기 YoY와 크게 불일치하거나 베이스가 비정상적으로 작으면(트로프 베이스효과) flag 추가("트로프 베이스효과 의심 — YoY 점수 미반영"). 단위·통화 일관성 assert.
(B) insight.py — 한 줄 인사이트 자동 생성: 기여도 최상위 레이어/컴포넌트를 골라 자연어 한 문장. 예: "3M 컨센 +78% 상향이 가속 중, YTD 런레이트가 연간 컨센 상회 → 추가 상향 여력. 단 YoY는 트로프 베이스효과로 미반영." flags 있으면 끝에 경고 덧붙임.
(C) compute_eps_revision_score() 메인 함수: 아래 스키마로 조립.
  { "eps_score": float, "layers": {"realized","momentum","forward"}, "confidence": float, "evidence": {...}, "insight": str, "flags": [...] }
통합 단위테스트: 정상케이스 1 + 트로프베이스케이스 1 + 저커버리지케이스 1. 모두 통과.
```

## 5. 반드시 지킬 주의사항 (실수 방지 체크리스트)

1. **YoY 절대 금지**: 점수 입력은 컨센 추정치 변화율(`now/3M전-1`)만. 전년 실적 대비 성장률은 베이스 효과로 왜곡됨.
2. **결측 처리**: 결측 컴포넌트는 `None` → 집계 시 **가중 0으로 빼고 재정규화**. 절대 0으로 채우지 말 것 (0은 "중립"이라는 잘못된 신호가 됨).
3. **자기상관계수는 외부 주입**: `persistence`의 섹터 자기상관계수를 모듈 내부에서 추정하지 말 것(룩어헤드 편향). 별도 캘리브레이션 산출물로 주입.
4. **신뢰도 게이트 필수**: 애널리스트 2~3명이 만든 점수를 풀 신뢰하면 노이즈에 휘둘림.
5. **섹터 상대화**: 모든 표준화는 섹터 내에서. 섹터 간 절대 비교 금지(섹터마다 리비전 변동성 다름).
6. **출력은 숫자 하나가 아니라 분해 + 인사이트 + 플래그**. 화면 '근거자료' 패널이 이 evidence/insight를 그대로 렌더.
